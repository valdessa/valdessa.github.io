🍗 Poyo Engine VX

PoyoVX is a multi-backend game engine and successor to Poyo Engine (2022–2023). The name says it all — V for Vulkan, X for GX (the GameCube graphics API).

Built from scratch out of a passion for computer graphics and engine development. Supporting both modern and legacy APIs is the challenge that drives this project — seeing the engine run on a Nintendo Switch, and eventually on a GameCube, is what keeps the motivation going.

Rendering Architecture

The engine uses a two-layer rendering architecture:

  • Low-level RHI: Vulkan 1.3 & OpenGL 4.6 — programmable shader pipeline
  • High-level RHI: Abstracts all 3 APIs including GameCube GX — a fixed-function pipeline with no shader support, requiring a completely different approach

Key Features

🔺

Vulkan 1.3

Full backend with VMA, synchronization2, and dynamic state.
🎮

Nintendo Switch

All demos running on Switch via EGL, touch input, and custom logger.
🎭

Deferred Shading

G-buffer, MRT, blend modes on both backends.
🌊

Shadows

Cascaded, point (cubemap), and dual paraboloid shadow maps.
🔷

Mesh Shaders

OpenGL (NV extension) and Vulkan (EXT) mesh shader support.
📐

Tessellation

GPU terrain generation on both backends.
🛠️

Editor

10 panels, theme system, shader editor with live reload.
🗂️

Scene System

ECS (entt), YAML serialisation, undo/redo action history.
💡

Lighting

Point, spot, area lights with PBR bloom (CoD:AW).
🔤

Text Rendering

2D, 3D, and SDF text via FreeType.
📦

KTX2 Textures

Compressed textures: ASTC, BC, ETC2.
🖥️

Compute Pipeline

Dispatch and indirect compute on both APIs.
