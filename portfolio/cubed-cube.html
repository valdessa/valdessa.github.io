An Optimised Voxel Engine for the Nintendo GameCube

This is my final individual university project, focused on creating a voxel engine optimized for the Nintendo GameCube from scratch using its native Graphic API Nintendo GX. The main goal was to maximize performance and memory efficiency to render the highest number of chunks while maintaining a stable 60 FPS on the console's limited hardware.

Optimizations in the Voxel Engine

To achieve a smooth 60 FPS while rendering as many chunks as possible, I focused on both performance and memory efficiency with these key optimizations:

  • Batching:

    • Reduced CPU overhead by combining multiple draw calls into one.

  • Occlusion Culling:

    • First culled entire cubes, then refined it to cull individual cube faces, saving more processing power by not rendering hidden faces.

  • Memory Optimization with Bitfields:

    • Used bitfields in structs to pack data efficiently, allowing more chunks to fit in memory without crashes.

  • Display Lists:

    • Pre-recorded draw commands in display lists to reduce draw calls and improve FPS, at the cost of some memory usage.

  • Frustum Culling:

    • Rendered only chunks within the camera’s view, further optimizing performance.

  • Dynamic Chunk Management:

    • Loaded and unloaded chunks dynamically around the camera, reducing memory use and improving load times.

  • Efficient Terrain Generation:

    • Optimized terrain generation to minimize redundant calculations and improve performance.
